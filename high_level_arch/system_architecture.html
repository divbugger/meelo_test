Key Data Flows & Communication Patterns:

1. User Authentication: Flutter → AuthService → Supabase Auth → PostgreSQL (profiles table)

2. Story Creation: User Input → StoryService → MistralService → Mistral API → Generated Story → ElevenLabsService → ElevenLabs API → Audio File → Supabase Storage

3. Real-time Synchronization: Supabase Real-time → StoryService → UI Updates (WebSocket)

4. Language Management: LanguageService ↔ SharedPreferences ↔ User Profile (Supabase)

5. Story Retrieval: StoryService → Supabase Database → Audio URLs → Supabase Storage

Technical Implementation Details:

Authentication: JWT tokens, email/password, row-level security (RLS)

Database: PostgreSQL with real-time subscriptions, upsert operations

File Storage: Public URLs for audio files, automatic cleanup on story deletion

AI Integration: HTTP REST APIs with error handling and timeout management

Localization: Multi-language support (English/German) with dynamic switching

State Management: Provider pattern for global state, ChangeNotifier for services
